# 01. プロジェクト概要

## 目的

厚生労働省が定める特定健康診査（特定健診）の標準的な電子的記録様式（標準規格3-1b）に基づくHL7 CDA形式のXMLファイルを、HL7 FHIR R4形式のJSONに変換するシステムを構築する。

## 背景

- 特定健診データは医療機関・健保組合間でHL7 CDA形式のXMLとして交換される
- 次世代の医療情報標準であるHL7 FHIRへの移行ニーズが高まっている
- FHIRサーバー（HAPI FHIR等）との連携を将来的に見据え、変換基盤を整備する

## スコープ

### 対象

- 入力: 特定健診XML（HL7 CDA R2、厚生労働省標準規格3-1b準拠）
- 出力: HL7 FHIR R4 JSON（Bundle形式）

### 対象外（将来対応）

- HAPI FHIRサーバーへの直接POST
- 特定健診以外のCDAドキュメント
- FHIR R5対応

## システム全体像

```
┌─────────────────────────────────────────────────────┐
│                   Docker環境                         │
│                                                     │
│  ┌───────────┐    ┌──────────────┐    ┌──────────┐  │
│  │  Watcher  │───▶│  Converter   │───▶│  Output  │  │
│  │(APScheduler)   │ (Python)     │    │(JSON出力)│  │
│  └───────────┘    └──────┬───────┘    └──────────┘  │
│                          │                          │
│                   ┌──────▼───────┐                  │
│                   │  PostgreSQL  │                  │
│                   │(ログ・状態管理)                  │
│                   └──────────────┘                  │
└─────────────────────────────────────────────────────┘

data/
├── input/    ← 変換対象XMLを格納
├── output/   ← FHIR JSONを出力
├── success/  ← 変換成功XMLを移動
├── error/    ← 変換失敗XMLを移動
└── logs/     ← エラーログを出力
```

## 非機能要件

| 項目 | 要件 |
|------|------|
| 実行環境 | Docker / Docker Compose |
| 言語 | Python 3.11+ |
| データベース | PostgreSQL |
| FHIRバージョン | R4 |
| 監視間隔 | 設定可能（デフォルト5分） |
